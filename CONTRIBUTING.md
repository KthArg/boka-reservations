# Contribuir a este proyecto

Este repositorio usa metodología **feature-driven + spec-driven** con reglas estrictas de colaboración.

Antes de tocar cualquier código, leer las skills en `.claude/skills/`:

- `feature-workflow` — el ciclo completo desde idea hasta merge. Punto de entrada obligatorio.
- `spec-authoring` — cómo escribir un spec antes de codear.
- `codebase-conventions` — reglas técnicas del repo (TypeScript, estructura, DB, estilos).
- `testing-practices` — prácticas de testing por criticidad.
- `commit-and-pr` — convenciones de commits, ramas y PRs.
- `changelog-maintenance` — registro continuo de trabajo por feature.
- `external-services-vetting` — verificación obligatoria antes de incorporar servicios externos.
- `project-memory` — memoria persistente del proyecto, leer al inicio de cada sesión.

Ninguna feature arranca sin spec aprobado. Ningún PR se mergea sin tests.
